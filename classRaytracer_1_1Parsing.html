Detailed Description

Handles the parsing of configuration files for setting up the raytracer components.

The Parsing class is responsible for reading and interpreting configuration files to initialize the camera, primitives, and lights used in the raytracer. It provides methods to parse and retrieve these components.

void Raytracer::Parsing::parseCamera ( )

Parses the camera configuration from the libconfig settings.

This function retrieves the camera settings from the configuration file, including resolution, position, rotation, and field of view (FOV). It then initializes a Camera object with the parsed values. If any required field is missing, an exception is thrown, and an error is logged.

Exceptions
libconfig::SettingNotFoundExceptionIf a required field is missing in the configuration.

The following fields are expected in the configuration:

  • camera:
    • resolution:
      • width (int): The width of the camera resolution.
      • height (int): The height of the camera resolution.
    • position:
      • x (int): The x-coordinate of the camera position.
      • y (int): The y-coordinate of the camera position.
      • z (int): The z-coordinate of the camera position.
    • rotation:
      • x (int): The x-axis rotation of the camera.
      • y (int): The y-axis rotation of the camera.
      • z (int): The z-axis rotation of the camera.
    • fieldOfView (float): The field of view of the camera.

void Raytracer::Parsing::parseLights ( )

parsing

Parses the light settings from the configuration file and initializes the corresponding light objects.

This function reads the "lights" section from the configuration file and extracts the following:

  • Ambient light intensity.
  • Diffuse light intensity.
  • Point lights with their positions.
  • Directional lights with their directions.

The parsed light objects are stored in the appropriate member variables:

  • ambientLight: Stores the ambient light object.
  • diffuseLight: Stores the diffuse light object.
  • lightsList: Stores a list of point and directional light objects.
Exceptions
libconfig::SettingNotFoundExceptionIf a required field is missing in the configuration file.
Note
Logs the parsing process and any errors encountered using the Logger class.

void Raytracer::Parsing::parsePrimitives ( )

Parses the primitives (spheres and planes) from the configuration file and adds them to the primitive list.

This function reads the "primitives" section of the configuration file, extracting information about spheres and planes. It creates the corresponding primitive objects, applies transformations, and stores them in the primList member.

Exceptions
std::invalid_argumentIf an invalid axis is specified for a plane.
libconfig::SettingNotFoundExceptionIf a required setting is missing in the configuration file.

The following primitives are supported:

  • Spheres: Defined by position (x, y, z), radius (r), and color (r, g, b).
  • Planes: Defined by axis (X, Y, Z), position, and color (r, g, b).

Example configuration structure:

primitives:
{
spheres = (
{ x = 0; y = 0; z = 0; r = 10; color = { r = 255; g = 0; b = 0; }; },
{ x = 5; y = 5; z = 5; r = 15; color = { r = 0; g = 255; b = 0; }; }
);
planes = (
{ axis = "X"; position = 10; color = { r = 0; g = 0; b = 255; }; },
{ axis = "Y"; position = -5; color = { r = 255; g = 255; b = 0; }; }
);
}
