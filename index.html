Raytracer

Description

The goal of this project is to develop a Raytracer application — a program designed to render 3D scenes using ray tracing techniques. Ray tracing simulates the way light interacts with objects to produce realistic images by calculating reflections, refractions, and shadows.

This project supports the following features:

  • Rendering of 3D objects with realistic lighting and shading.
  • Support for multiple object types such as spheres, planes, and cubes.
  • Configurable materials, including reflective and transparent surfaces.
  • Dynamic scene configuration through external files.

Requirements

  • C++ Compiler
  • libconfig++ library (sudo apt-get install libconfig++-dev)
  • SFML library (sudo apt install libsfml-dev)

Usage

make
./raytracer FILE.cfg

Design patterns

  1. Factory Pattern Où : Dans la méthode Parsing::parsePrimitives() et Parsing::parseLights(). Explication : Ont utilise des appels comme std::make_shared<Sphere> ou std::make_shared<Plane> pour créer des objets de primitives ou de lumières. Cela correspond à une implémentation simple du pattern Factory, où la création d'objets est centralisée et abstraite.
  2. Composite Pattern Où : Dans la gestion des primitives et des lumières. Explication : Les primitives (Sphere, Plane, etc.) et les lumières (AmbientLight, PointLight, etc.) sont toutes dérivées d'interfaces communes (IPrimitive et ILights). Cela permet de traiter un groupe d'objets comme une seule entité, ce qui est une caractéristique clé du pattern Composite.
  3. Observer Pattern Où : Dans la classe Logger. Explication : Le Logger agit comme un observateur qui enregistre les événements ou messages générés par d'autres parties du système (comme Parsing ou Renderer). Cela correspond au pattern Observer, où un objet est notifié des changements ou événements dans d'autres objets.
  4. State Pattern Où : Potentiellement dans la gestion des états de rendu ou des transformations. Explication : Si le système de rendu ou les objets (comme Plane ou Sphere) changent de comportement en fonction de leur état (par exemple, transformations appliquées via Matrix4x4), cela peut être vu comme une utilisation du pattern State.
